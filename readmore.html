Semantic Pathfinding

Semantic pathfinding is an algorithm I developed in my research, alongside a team of six under the Rutgers Aresty Undergrad Research Program. It's created by combining the techniques of hierarchical pathfinding, A*, and influence maps.

The image below is the virtual environment made in Unity used for the study. It's a model of a hospital building. Each colored area is considered a "zone." I used zones to create higher level searches for A*, which result in hierarchical pathfinding.



My case study was crowd congestion, which is displayed in this image below. An influence map is placed in the environment to quantify crowds in the environment. With this map, new values are added to each agent's pathfinding calculations. Visually, this makes the environment look like a heat map, with red areas being more favorable. The result is a pathfinding algorithm that not only finds the shortest path, but one that also considers the dynamic stimuli of the agents, like noise, crowds, visibility, etc. The disadvantages of this implementation is efficiency though, since the pathfinding algorithm re-calculates paths for the entire environment at each frame.



In retrospective, this was my first experience working with a large team in a professional setting! The study was an overall success, despite the efficiency errors in the code. I not became more proficient with Unity and C#, but I also learned soft skills pivotal for any leader or team player. ü§ó
